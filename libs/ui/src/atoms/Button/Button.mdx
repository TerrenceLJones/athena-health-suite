# Button

A single clickable action with five variants (`primary`, `secondary`, `ghost`, `danger`, `link`),
three sizes, and disabled/loading states that are visually and behaviorally distinct. `link`
renders as bare inline text — no padding, border, or background — for secondary actions that
should read as a text link rather than a button (e.g. "Back to sign in").

## User story

**US-AP-012 — Provider UI atom library**

As an Athena Provider engineer building any feature epic, I want the 11 Clearline-sourced UI
atoms (Button, TextField, PasswordField, Text, Container, Checkbox, Alert, Chip, Avatar,
ProgressBar, SegmentedControl) ported wholesale and re-themed to Athena's own tokens and icons,
so that I never hand-roll a button, field, or badge and risk missing an accessibility rule the
design system already solved.

## Acceptance criteria

- **AC-01 — Button.** Given Button with `loading={true}`, when it renders, then the button is
  disabled, shows a spinning icon, and does not fire `onClick`.
- AC-02 — TextField. Given TextField with `state="error"` and an `error` message, when it
  renders, then the input has `aria-invalid="true"` and `aria-describedby` pointing at a
  `role="alert"` element containing the error text.
- AC-03 — Checkbox. Given Checkbox (wrapping Radix `Checkbox`), when a user clicks it or
  activates it via keyboard (Space), then `onCheckedChange` fires and `aria-checked` reflects
  checked/indeterminate/unchecked correctly.
- AC-04 — PasswordField. Given PasswordField's visibility toggle, when activated, then the
  input's `type` switches between `password` and `text` without losing focus or cursor position.
- AC-05 — SegmentedControl. Given SegmentedControl, when used controlled (`value`/`onChange`) or
  uncontrolled (internal state), then it tracks and reports the active option correctly in both
  modes.
- AC-06 — Text. Given `Text` with a given `size`, when it renders, then font size, weight
  default, letter-spacing, and font family all come from the token pipeline's `typography.scale`
  for that step, independent of its `as` prop (e.g. `size="body"` `as="h3"`).

### Edge case covered here

Disabled and loading are both exercised as separate stories — both block `onClick` via the same
`aria-disabled` + guarded-click-handler mechanism and stay focusable, but only loading shows the
spinner and `aria-busy`, so the two states must never be conflated.
